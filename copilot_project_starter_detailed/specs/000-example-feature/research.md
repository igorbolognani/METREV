# Research Notes — Example Feature

## Goal
Provide lightweight notes showing where research would live for a real feature.

## Questions
- Is a full feature spec needed for every change?
- Which project sizes benefit most from a spec-first approach?
- Which docs must always be updated with architecture changes?

## Preliminary guidance
- Small bugfixes may only need a short plan and tests.
- Medium and large features benefit from spec -> plan -> tasks.
- If architecture, contracts, or onboarding change, docs should be updated.
