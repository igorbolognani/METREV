# Feature Specification — Example Feature

## Objective
Provide an example feature folder to demonstrate the spec-first workflow.

## Why
This gives the team and the coding agent a concrete reference of how specs should look.

## Primary users
Developers, reviewers, and AI agents working in this repository.

## Scope
### In
- example structure
- example requirements
- example acceptance criteria

### Out
- production-ready implementation
- framework-specific code

## Functional requirements
1. The repository must include a reusable feature specification template.
2. The repository must include a matching plan and tasks template.
3. The example must be simple enough to copy for a real feature.

## Acceptance criteria
- [ ] A developer can duplicate this folder for a new feature.
- [ ] The example shows how spec, plan, and tasks relate.
- [ ] The example does not assume a specific language or framework.

## Risks / unknowns
- Teams may skip specs for small changes.
- Teams may overcomplicate specs if no size threshold is defined.
