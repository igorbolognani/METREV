# Acceptance Checklist

## Functional
- [ ] behavior matches the spec
- [ ] acceptance criteria are satisfied
- [ ] failure cases behave as expected

## Technical
- [ ] tests pass
- [ ] lint/format pass
- [ ] no obvious architecture drift
- [ ] no unnecessary dependency added

## Safety
- [ ] inputs validated
- [ ] auth/permissions reviewed if relevant
- [ ] error handling present
- [ ] no sensitive data exposure

## Documentation
- [ ] spec updated if needed
- [ ] ADR updated if needed
- [ ] quickstart/checklist updated if needed
