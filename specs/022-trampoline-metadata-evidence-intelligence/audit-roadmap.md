# Audit And Roadmap - TRAMPOLINe Alignment

## Current Alignment

METREV already matches the core TRAMPOLINe thesis: industrial adoption of microbial electrochemical technologies needs more than papers and prototypes. The repository has MFC, MEC, and broader MET vocabulary, canonical domain/contract boundaries, research review tables, evidence packs, review gates, lineage, report-grounded conversation, and deterministic confidence disclosure.

## Main Gaps Closed By 022

- Metadata quality becomes a first-class confidence input instead of a loose note inside payloads.
- Local PDFs become importable source artifacts with file hash, page/chunk locators, and reviewable extraction outputs.
- TRAMPOLINe becomes a curated ecosystem map for industrial-adoption vocabulary rather than an informal external reference.
- Veracity components make the Big Data problem explicit: volume and velocity are not enough without variety, metadata, traceability, and truthfulness.

## Professional-Tool Posture

The platform remains decision support, not a simulator-first product. The professional value is auditability: observed input, normalized input, missing data, defaults, evidence, scoring, uncertainty, and next measurements must remain separated. TRAMPOLINe context improves ontology and evidence targeting, but every recommendation still depends on reviewed evidence and explicit confidence.

## Roadmap

1. Stabilize metadata and veracity contracts.
2. Import the TRAMPOLINe curated map and the three water metadata PDFs.
3. Use local PDF artifacts to create review tables and evidence packs.
4. Extend benchmark extraction only after enough reviewed source chunks exist.
5. Promote recurring metadata-quality checks into evaluation/report readiness.
